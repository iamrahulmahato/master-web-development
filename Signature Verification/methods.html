Methods Used in Signature Verification

This Signature Verification System uses several steps to analyze and compare the signatures. Below are the main steps involved:

1. Preprocessing the Signatures

The first step is to preprocess the uploaded images. The following preprocessing techniques are applied:

  • Grayscale Conversion: The color images are converted into grayscale to reduce complexity.
  • Thresholding: Thresholding is applied to the image to create a binary image that distinguishes the signature from the background.
  • Contour Detection: Contours are detected in both the uploaded signatures using OpenCV to extract the structure of the signatures.

2. Comparing Signatures

The core method of comparison involves calculating a similarity score between the two signatures:

  • Histogram Comparison: We use OpenCV’s cv2.compareHist() function to compare the histograms of the two signatures. This gives a score indicating the degree of similarity between the two images.
  • Correlation Method: The correlation method is used to compute the similarity between the two histograms. A higher correlation score means the signatures are more similar.

3. Decision Making

Once the similarity score is calculated, a threshold is applied to determine if the signatures match:

  • If the similarity score is above the predefined threshold, the signatures are considered to be from the same person.
  • If the score is below the threshold, the signatures are likely different.

Technologies Used

  • Flask: Flask is used to handle the backend logic and serve the web application.
  • OpenCV: OpenCV is used for image processing and signature comparison.
  • Jinja2: Flask’s templating engine Jinja2 is used to render dynamic HTML pages.
